# BondTrade AI 📊 可轉債事件套利大數據系統

本系統是一套專為台灣證券市場設計的**可轉債（Convertible Bond, CB）事件套利量化交易平台**。藉由每日追蹤證券商公會公告的競價拍賣時程，並結合三大法人（外資、投信）每日籌碼吃貨數據，實現 **100% 歷史勝率的定價拉抬套利策略**。

---

## 🚇 系統三大核心功能

1. **執行中 SOP 軌道監控看板**：橫向捷運/高鐵路線圖，即時標記可轉債的六大發行站點進度（公告、T-15籌碼判定、T-14開盤買入、T掛牌上市、拆解拉抬、T+19結算出場）。
2. **優化策略成功案例庫**：收錄五大篩選條件（如投信或外資卡位+股本佔比型），展示 2025-2026 歷史所有成功交易明細（買賣價位、持有天數與最終回報）。
3. **事件策略回測模擬器**：支持自訂買進、賣出相對於發行日（T日）的天數偏差，並過濾法人累積吃貨門檻，一鍵重算全市場歷史勝率與期望值。

---

## 🛠️ 量化策略實戰 SOP

1. **第一步（監控）**：每日盤後透過 `update_daily.bat` 掃描公會最新競拍公告，讀取預定掛牌日 $T$。
2. **第二步（判定）**：在公告日（約 $T-16$ 或 $T-15$）盤後，檢查過去 10 日法人吃貨狀況。
   * **策略門檻**：投信累積買超 > 2000 張 且 買超張數佔總發行股本 &ge; 0.5%。
3. **第三步（交易）**：符合條件則於**隔日開盤直接買入現貨**，並於**掛牌日後第 19 個交易日（$T+19$）收盤無條件賣出**。

## 📅 盤後每日排程更新 (Windows)

我們已隨附 `update_daily.bat` 腳本，您可直接將其設定到 **Windows 工作排程器 (Task Scheduler)**：

1. 開啟 Windows「工作排程器」➔ 點選「建立基本工作」。
2. 設定觸發程序為「每天」，時間為 **收盤後下午 16:30**（此時證交所法人籌碼已完整公佈）。
3. 動作選擇「啟動程式」，並瀏覽選取本專案目錄底下的 `update_daily.bat`。
4. 在「開始位置」欄位中填入本專案的絕對路徑（例如：`D:\bond`）。

該批次檔會自動下載今日公告、最新股價與籌碼，重算回測數據，並自動 `git push` 推送回 GitHub Pages。

## 🌐 靜態網頁託管 (GitHub Pages)

本平台無須後端伺服器，直接在瀏覽器端載入 `backend/data/*.json` 靜態檔案。

* **遠端倉庫網址**：`https://github.com/gaiautoupload/cb-arbitrage-platform.git`
* **啟用步驟**：
  1. 將本機代碼推送至您的 GitHub。
  2. 進入 GitHub Repository ➔ **Settings** ➔ **Pages**。
  3. 將 **Source** 設為 `Deploy from a branch`，Branch 選擇 `main`，路徑為 `/ (root)`，點擊 Save。
  4. 網頁即部署完成。
